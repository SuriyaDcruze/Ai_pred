# Release v0.1.0 — Forward Testing

- **Tag:** `v0.1.0-forward-testing`
- **Date:** 2026-07
- **Sprint:** 1 (COMPLETE)
- **Highlights:** the Forward Testing Engine — Aegis' first step from *backtest-only* toward
  a *real, logged, live track record*.

This is the first production-ready architectural milestone for Aegis. It adds the machinery
to record every actionable recommendation, resolve it against real future price, and report
the result **honestly** — with sample size and confidence, never over-claiming. It changes
**nothing** about how predictions are made.

## Features
- **Forward Testing Engine** (`app/forward_testing/`): records BUY/SELL recommendations,
  resolves them against future candles (target / stop / expiry, stop-first pessimistic),
  and maintains realised-R statistics. Restart-safe and idempotent.
- **Persistence** (`data/prediction_history.db`): the permanent `predictions` table with
  rich context and three independent version stamps (prediction model, outcome model,
  features), created by an append-only, versioned, idempotent migration. Creation fields are
  immutable; only lifecycle columns change on resolve.
- **REST API** (`/forward/*`): `POST /prediction`, `GET /prediction/{id}`, `/active`,
  `/completed`, `/stats`, `/summary`, `/breakdown`. Request validation, honest sample-size
  confidence, and a live-vs-backtest comparison. Full reference:
  [../api/forward-testing.md](../api/forward-testing.md).
- **Dashboard** (`/dashboard/forward.html`): a presentation-only page over the API — six
  sections (Overview, Live vs Backtest, Performance Breakdown, Active, Completed, Timeline),
  with loading / empty / error states and sample size shown everywhere. Linked from the main
  dashboard.
- **Background monitor** (`ForwardTestingMonitor`): an async resolution loop, implemented and
  tested (not yet wired into the running app — see Known limitations).

## Improvements
- **Honesty enforced in code:** `no_data` / `insufficient_sample` / `building` on
  `/forward/summary`, and `no_data` / `building_sample` / `inconclusive` /
  `statistically_significant` (95% CI) on live-vs-backtest.
- **Architecture Decision Records** introduced ([../architecture/adr/](../architecture/adr/)):
  modular monolith, immutable engines, forward-testing-before-production, single database,
  API/engine separation.
- **Documentation:** Volumes 18 & 20 updated to as-built; full API reference; Sprint 1
  report; this release note.
- **Testing:** 128 forward-testing tests added; full suite **361 passed**. Import-guard tests
  prove the API imports neither engine; all tests use temporary databases.

## Breaking changes
**None.** All changes are additive:
- New package `app/forward_testing/`, new API router `/forward/*`, new static page.
- New settings (`AEGIS_FORWARD_BACKTEST_*`) with safe defaults.
- New database `data/prediction_history.db` (created lazily); the legacy `calls.db` and all
  existing endpoints are untouched. The Prediction, Outcome, and Risk engines are unchanged.

## Known limitations
- **The edge is still backtest-only** — Forward Testing is the instrument; a live sample has
  not yet accumulated. No real-money claim until 50–100+ resolved live trades (ADR 0004).
- **No auto-record path yet** — predictions are recorded via `POST /forward/prediction`.
- **Monitor not wired into the app lifespan** — exists and is tested; needs a live
  candle-fetcher + config gate to run in the server.
- **"Current R" for open trades is unrealised** (`—/open`); booked only at target/stop/expiry.
- **SQLite single-writer** (mitigated by WAL + a lock); Postgres is the future path (Vol 21).
- **Backtest baseline is one configured constant** (crypto WF 59.6%), not per-market.

## Upgrade notes
No migration action required. On first run the app creates/migrates
`data/prediction_history.db` automatically. Optional environment overrides:
`AEGIS_FORWARD_BACKTEST_WIN_RATE`, `AEGIS_FORWARD_BACKTEST_AVG_R`,
`AEGIS_FORWARD_BACKTEST_PROFIT_FACTOR`, `AEGIS_FORWARD_BACKTEST_LABEL` (set win rate < 0 to
declare "no baseline configured").
